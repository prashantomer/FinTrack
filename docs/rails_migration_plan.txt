FINTRACK: RAILS MIGRATION PLAN
FastAPI/Python → Ruby on Rails
===============================

CONTEXT
-------
FinTrack backend is currently a FastAPI/Python monolith serving REST API at /api/v1/.
Goal: Replace with a Ruby on Rails API-only app using a fresh PostgreSQL database.
Frontend stays unchanged. Python backend archived as backend_python/.


PROS & CONS
-----------

PROS:
  + Convention over configuration — Rails generators reduce boilerplate vs FastAPI wiring
  + ActiveRecord — callbacks (after_create), scopes, audited gem replace SQLAlchemy event listeners
  + Rake tasks — built into Rails; replaces Typer CLI with no external dependency
  + Rails console — replaces `uv run python -m app.cli console`, richer REPL
  + Ecosystem — jwt, rack-cors, kaminari, audited, rspec all well-maintained and opinionated
  + Migrations — simpler than Alembic; datetime-stamp workaround in alembic/env.py gone
  + Test ergonomics — RSpec + FactoryBot > pytest + httpx AsyncClient setup

CONS:
  - Full rewrite risk: ~3,000+ lines of Python across 12 services, 12 routers, 13 models
  - No async: FastAPI is ASGI/async; Rails synchronous by default (instrument fetch blocks thread)
  - Loss of Pydantic validation: Rails needs explicit params.permit + model validations, no auto OpenAPI docs
  - Learning curve: slower feature velocity during transition
  - audited gem vs custom AuditLog table: audited creates own `audits` table — old audit_logs data lost
    unless a custom audit observer is written to keep the old table
  - No tangible user-facing benefit: API surface, frontend, DB structure stay the same

RECOMMENDATION:
  Proceed — the DB drop+recreate removes the biggest enum compatibility risk.
  Two high-risk areas to integration-test first:
    1. STI models (Investment, TermAccount) — Rails STI uses `type` string column; subclasses need
       careful mapping to preserve query behavior
    2. Balance-update callback chain — after_create on Transaction must exactly replicate
       _apply_balance_delta or account balances drift silently

PHASE 3 — AUTHENTICATION
--------------------------
  Python: python-jose (JWT) + passlib (bcrypt)
  Rails:  has_secure_password + jwt gem, no Devise

  lib/json_web_token.rb
    module JsonWebToken
      SECRET = Rails.application.credentials.secret_key_base
      EXPIRY = 7.days

      def self.encode(payload)
        JWT.encode(payload.merge(exp: EXPIRY.from_now.to_i), SECRET, 'HS256')
      end

      def self.decode(token)
        JWT.decode(token, SECRET, true, algorithm: 'HS256').first
      rescue JWT::DecodeError
        nil
      end
    end

  app/controllers/concerns/authenticatable.rb
    → current_user method
    → authenticate_user! before_action (returns 401 if missing/invalid token)

  Endpoints preserved exactly:
    POST /api/v1/auth/login
    GET  /api/v1/auth/me
    PUT  /api/v1/auth/me

      namespace :v1 do
        post 'auth/login', to: 'auth#login'
        get  'auth/me',    to: 'auth#me'
        put  'auth/me',    to: 'auth#update_me'
        resources :banks, only: [:index, :show]
        resources :accounts do
          member { post :close }
        end
        resources :transactions, only: [:index, :create]
        resources :investments
        resources :instruments do
          collection { get :types; get :tracked }
          member { post :track; delete :untrack }
        end
        resources :term_accounts, only: [:index, :show, :create] do
          member { post :close }
        end
        resources :platforms, only: [:index, :show]
        resources :platform_accounts
        resources :follios
        namespace :reports do
          get :dashboard
          get :spending_trends
          get :investment_summary
        end
        post 'errors', to: 'client_errors#create'
      end
    end

  Balance callback (replaces _apply_balance_delta):
    # app/models/transaction.rb
    after_create :apply_balance_delta

    private

    def apply_balance_delta
      # FD term accounts skip balance update (principal-based, not running balance)
      return if linked_account_type == 'term_account' && linked_account.sub_type == 'fd'
      delta = credit? ? amount : -amount
      linked_account.increment!(:balance, delta)
    end

PHASE 8 — DATABASE: DROP & RECREATE
--------------------------------------
  The existing fintrack_db is DROPPED and recreated fresh from Rails migrations.
  This eliminates all PostgreSQL native enum type baggage.

  Commands:
    dropdb fintrack_db
    createdb fintrack_db
    psql -c "GRANT ALL ON DATABASE fintrack_db TO fintrack_user;"
    cd backend && bundle exec rails db:migrate
    bundle exec rake db:seed:banks
    bundle exec rake db:seed:platforms

  All 6 PG native enums → Rails string enums:
    # No CREATE TYPE in DB — stored as varchar column
    enum :transaction_type, { credit: 'credit', debit: 'debit' }, validate: true
    enum :investment_type, { stock: 'stock', mutual_fund: 'mutual_fund', ... }, validate: true

  Column type mapping:
    Python/SQLAlchemy                     Rails Migration
    -----------------                     ---------------
    ARRAY(Text) for tags                  t.string :tags, array: true, default: []
    UUID for public_id                    t.uuid :public_id, default: -> { "gen_random_uuid()" }
    Enum (6 PG types)                     t.string :field_name, null: false  +  Rails enum on model
    server_default: func.now()            t.timestamps  OR  default: -> { "NOW()" }
    ondelete: "CASCADE"                   foreign_key: true, on_delete: :cascade
    UniqueConstraint([col1, col2])        add_index :table, [:col1, :col2], unique: true

  Migration order (FK dependency order):
    1.  users
    2.  banks, platforms
    3.  accounts (→ users, banks)
    4.  platform_accounts (→ users, platforms)
    5.  instruments
    6.  user_instruments (→ users, instruments)
    7.  term_accounts (→ users, accounts)
    8.  investments (→ users, user_instruments, platform_accounts)
    9.  transactions (→ users; linked_account polymorphic = no FK)
    10. follios (→ user_instruments)
    11. audits (auto-created by: rails generate audited:install)
